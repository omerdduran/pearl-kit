## [Unreleased]

### Added

- `PearlKit.Slider` — single-thumb continuous value selector with horizontal + vertical orientation and optional tick marks. Built on `QtQuick.Templates.Slider` with no `contentItem` override (T.Slider does not use one) and two custom slots: `background` holds the 6 px `rounded-full` track (`Tokens.muted`), a nested range `Rectangle` bound to `control.position` (`Tokens.primary`), and an opt-in tick `Repeater`; `handle` is a 16 × 16 `#FFFFFF` circle with a 1 px `Tokens.primary` border. Orientation-aware bindings swap the thin / long axes manually (T.Slider does **not** swap geometry on orientation flip). Horizontal defaults to 200 × 20, vertical to 20 × 176 (shadcn `min-h-44`). Vertical fill is **bottom-to-top** — lower values at the bottom of the track — matching iOS / macOS volume sliders and natural expectations for medical imaging window / level scales. Ticks are opt-in via `showTicks: bool` with `tickCount: int` (0 = auto from `stepSize`, capped at 21), rendered 1 × 4 px in `Tokens.mutedForeground` on the far side of the track. Thumb scales to 1.05× on `control.pressed` via `Behavior on scale` (`Tokens.motion.fast`, `Easing.OutCubic`). Keyboard navigation is native T.Slider (Left / Right / Up / Down / PageUp / PageDown / Home / End — no custom `Keys.on*Pressed` handlers). Focus ring 4 px @ `ring / 50 %`, `offset: 0`, keyboard-only via `visualFocus`, targeting `control.handle` so the ring follows the thumb. Disabled state fades track + handle to 50 % alpha without touching the root control (standard pearl-kit rule — keeps the focus ring crisp). Deliberate divergences from shadcn: no hover ring on the thumb (shadcn shows `ring-4` on both hover and focus-visible; 16 px thumb with constant pointer motion created visual noise — Button / Toggle convention wins); pearl-kit-specific `showTicks` + `tickCount` extension (shadcn Slider ships no ticks) for window / level and zoom controls; single-thumb only (two-thumb range selection would require `T.RangeSlider` and is deferred); no thumb `shadow-sm` (Toggle precedent — negligible on a control this small).
- `PearlKit.Toaster` / `PearlKit.ToasterHost` / `PearlKit.Toast` — Sonner-style notification system. `Toaster` is a `pragma Singleton` QML singleton exposing `show({type, title, description, duration})` (returns an integer `id`), `dismiss(id)`, and `dismissAll()`, plus configurable `position` (six variants, default `top-right`), `maxStack` (default `3`), and `defaultDuration` (default `4000` ms). `ToasterHost` is the renderer piece — a plain `Item` placed once inside `ApplicationWindow` that auto-reparents to `T.Overlay.overlay`, subscribes to `Toaster._stack`, and renders each row via a `Column` + `Repeater` with `add` / `move` transitions for stack animation. `Toast` is the delegate surface: `Item` + `Rectangle` (intentionally not `T.Popup` — stacked popups create z-order and modal conflicts), 356 px wide, `Tokens.popover` / `Tokens.popoverForeground` / `Tokens.border` / `Tokens.radius.md` mapped from Sonner's CSS variable exports, two-layer shadow via `MultiEffect`, type-colored icon (`PearlToastIcon` for success / info / warning / error, public `Spinner` reused for loading), `PearlBaseText` title (semibold) + description (regular), and a close button that mirrors `Dialog`'s close pattern (2 px `PearlFocusRing` @ `ring / 50 %`, Space / Return activation). Enter animation: 40 px slide + fade in at `Tokens.motion.base` (180 ms, `Easing.OutCubic`); exit is two-phase so adjacent toasts don't jump while the exiting row animates out. Hover pauses the auto-dismiss `Timer`; loading toasts default `duration: 0` and must be dismissed explicitly. Ships with `pearl_kit.NotificationCenter` + opt-in `pearl_kit.register_notifications(engine, icon_path=None)` — a Python `QObject` that wraps `QSystemTrayIcon` and exposes `isAppActive` (reactive via `applicationStateChanged`) + `showOsNotification(title, body, is_error)`; registered as a `NotificationCenter` context property so `ToasterHost` can probe for it at load time and route show calls to the OS tray when the app is backgrounded (replaces `airontgen`'s `NotificationManager` with the same focus-gated split). Deliberate divergences from Sonner: no swipe-to-dismiss, no action buttons, no rich content slot, no `promise()` API, hover-resume restarts the timer from zero instead of preserving remaining time. New `PearlKit.internal.PearlToastIcon` atom — 20 × 20 Canvas glyph dispatcher for success / info / warning / error (loading reuses public `Spinner`), following the `PearlChevron` / `PearlXIcon` / `PearlCheckIcon` pattern (no SVG assets, no `QtSvg` plugin dependency).
- `PearlKit.Tooltip` — seventeenth public component. Small transient hover overlay built on `QtQuick.Templates.ToolTip` with shadcn/ui v4 TooltipContent styling: inverted surface (`bg-foreground` → `Tokens.foreground`), inverted text (`text-background` → `Tokens.background`), `Tokens.radius.md` (6 px), 12 × 6 px padding, 12 px `text-xs` via `Tokens.font.size.xs`. Properties: `placement: "top" | "bottom" | "left" | "right"` (default `"top"`, no auto-flip — consumer picks the right side per trigger), `maxWidth: int` (default 320; long text wraps at this column via `Text.WordWrap` in concert with `implicitWidth: Math.min(implicitContentWidth + leftPadding + rightPadding, maxWidth)` and `contentItem.width: control.availableWidth`), `delay: 0` default matching shadcn/Radix `TooltipProvider` parity, `timeout: -1` (no auto-hide — desktop convention). `contentItem` overridden with `PearlBaseText` so font family / weight / `Text.NativeRendering` match the rest of pearl-kit typography; `background` overridden with a plain `Rectangle` at `Tokens.foreground` + `Tokens.radius.md`. Enter / exit run parallel `ParallelAnimation` fade (0 → 1) + scale (0.95 → 1) over `Tokens.motion.fast` (150 ms, Tailwind `animate-in` parity) with `Easing.OutCubic`. Positioning is explicit `x` / `y` computed from `parent.width` / `parent.height` per placement, flush against the trigger edge (shadcn `sideOffset: 0`). `closePolicy: T.Popup.NoAutoClose` so the tooltip only closes when the `visible` binding flips, never on outside click or Escape. `Accessible.role: ToolTip` with `Accessible.name` bound to `text` for VoiceOver / screen-reader support. No focus ring — tooltips don't receive focus (`focus: false`, the Popup default). Consumer wires visibility explicitly via `visible: trigger.hovered && trigger.enabled` — pure-QML idiom, no `QQuickStyle` attached-property plumbing. Deliberate divergences from shadcn: no arrow (macOS native convention, non-deterministic without auto-flip — deferred); no auto-flip placement (fixed 4-direction, consumer picks side — `FloatingUI`-style flip deferred to v0.4.0+); no enter-animation slide-in-from-opposite (visual delta imperceptible at 150 ms, keeps enter transition single-form across placements); no `sideOffset` property (hardcoded flush = shadcn source default; consumers override `x` / `y` for custom gap); no Qt attached `ToolTip.text` syntax (requires `QQuickStyle` C++ plugin — pearl-kit stays pure-QML, standalone `P.Tooltip { }` is the only entry point); no `TooltipProvider` / portal / state-machine layers (QML-idiomatic: one standalone popup per trigger, visibility consumer-wired).
- `PearlKit.MessageBox` — sixteenth public component. Opinionated modal-message primitive built as a QML subclass of `PearlKit.Dialog`, with four `variant` values (`info` / `warning` / `error` / `confirm`) driving both an auto-generated tinted icon badge and the footer button set. Properties: `variant: string`, `heading: string`, `message: string`, `okText: string` (default `"OK"`), `cancelText: string` (default `"Cancel"`), `okVariant: string` (defaults to `"destructive"` when `variant === "error"`, else `"default"`; consumer-overridable). Signals: `accepted()` (OK clicked / Enter on focused OK) and `rejected()` (Cancel clicked, or Escape / overlay dismiss for `confirm` variant only). MessageBox suppresses Dialog's native header (`title: ""`, `description: ""`) and renders its own `icon + heading + message` `ColumnLayout` inside the inherited `_bodyContent` default slot; the footer is auto-built as a right-aligned `Row` with a Button (`okVariant`) always present and an outline Cancel Button visible only when `variant === "confirm"`. `showCloseButton: false` and `maxWidth: 420` hardcoded. For the `confirm` variant, `closePolicy` downgrades to `T.Popup.CloseOnEscape` only (overlay-click dismiss blocked, PyQt6 `QMessageBox` idiom) and an internal `QtObject { property bool resolved }` flag tracks button-driven vs. Escape-driven close so `onClosed` emits `rejected()` iff the user Escaped without picking. `initialFocusItem: okButton` auto-focuses OK on open so Enter immediately accepts. Deliberate divergences from shadcn's AlertDialog (12 primitives collapsed to one component, 4-variant taxonomy extending shadcn's `default`/`destructive`, variant-tinted badge at 15% alpha vs. shadcn's neutral `bg-muted` `AlertDialogMedia`, auto-generated footer instead of opt-in `AlertDialogAction` / `AlertDialogCancel`, `confirm` blocking overlay-click, narrower default `maxWidth: 420` vs shadcn `sm:max-w-lg` 512, no close X button).
- `PearlKit.internal.PearlAlertIcon` — Canvas-based tinted-badge atom with per-variant glyph (lowercase `i` for info, `!` for warning, `×` for error, `?` for confirm). 48×48 `Item` with a `Tokens.radius.md` `Rectangle` background at variant-tinted color (primary / warning / destructive @ 15 % α, or neutral `Tokens.muted` for confirm) and a centered 24×24 Canvas glyph stroked in the full-alpha variant color. Mirrors the `PearlXIcon` / `PearlCheckIcon` / `PearlChevron` pattern — no SVG assets, no Qt5Compat dependency. Repaint triggered via `Connections { target: root; function onVariantChanged() }` so glyph re-renders when the parent switches variants at runtime.
- `PearlKit.StatusBar` — fifteenth public component. Desktop main-window footer primitive with three regions: left-aligned hint, absolutely centered measurement, right-aligned status. Four `statusKind` values (`default` / `success` / `warning` / `error`) drive a derived `readonly property color statusColor` that consumers bind from the right slot for semantic coloring (mutedForeground / success / warning / destructive). Root is a plain `Rectangle` (no `QtQuick.Templates` base — a status bar has no interactive contract) filled with `Tokens.muted` and capped by a 1 px top hairline in `Tokens.border`. Three `property Component` slots (`leftContent`, `centerContent`, `rightContent`) instantiated through sibling `Loader`s anchored to the left edge, absolute horizontal center, and right edge respectively — absolute centering (not flexbox spacing) keeps the measurement fixed in the middle of the window even when left/right widths diverge (macOS Finder / Xcode convention). `hpad: int` (default 12, `Tokens.space.x3`) controls the outer slots' side inset. Implicit size 28 × 480 px — height matches the macOS footer idiom (between Xcode 23 px and Finder 24 px with slightly more breathing room for 14 px body text callers), width is a placeholder intended to be overridden via `Layout.fillWidth: true` or `anchors.left`/`anchors.right`. Disabled state fades each Loader independently to 50 % alpha without touching the root control — preserves child atoms (e.g. a focus ring around a clickable hint) from parent-opacity inheritance. Background color transition animated at `Tokens.motion.fast` (150 ms) for theme-switch smoothness. No shadcn/ui equivalent — shadcn targets web-first layouts where persistent footers are rare; design language is taken from the macOS footer idiom and documented as a deliberate divergence.
- `PearlKit.Spinner` — fourteenth public component. Small indeterminate activity indicator for inline loading states — a 3/4 circle arc that rotates continuously at 1 revolution per second (shadcn `animate-spin` parity). Root is a plain `Item` (no `QtQuick.Templates` base — spinners have no interactive contract). Arc rendered through a child `Canvas` using `ctx.arc` with `lineCap: "round"` (mirrors lucide `Loader2Icon`'s round stroke caps), sweeping 270° from top (−π/2) for a clean symmetric gap; `strokeWidth: 2` default matches lucide's stroke-2 at viewBox-24. Rotation driven by `RotationAnimator on rotation` attached to the Canvas — scenegraph-thread animation, so the Canvas paints once and only the transform ticks per frame. Properties: `color: Tokens.mutedForeground` (theme-reactive via Canvas `Connections` → `requestPaint()`), `strokeWidth: 2`, `running: true`, `duration: 1000` ms. Implicit size 16 × 16 (shadcn `size-4`); consumers resize via `width` / `height` directly — no string size scale. `running` gated by `control.visible` so hidden spinners don't burn scenegraph time. Accessibility via `Accessible.role: Accessible.Indicator` with `Accessible.name: "Loading"` (closest Qt equivalent to shadcn's `role="status"` + `aria-label="Loading"`). Deliberate divergences from shadcn: 270° arc instead of lucide's ~288° (visually imperceptible at 16–24 px, cleaner geometry); `running` and `duration` exposed (shadcn spins unconditionally at a fixed 1 s); explicit `color` property (QML has no CSS `currentColor` equivalent); no string size scale — `width` / `height` overrides cover every observed use case.
- `PearlKit.ProgressBar` — thirteenth public component. Thin pill-shaped horizontal progress indicator with both determinate and indeterminate modes. Defaults: `from: 0`, `to: 100`, `value: 0`, `implicitWidth: 200`, `implicitHeight: 8` (shadcn `h-2` parity); `to: 100` overrides `T.ProgressBar`'s native `to: 1` for a user-friendlier "45 means 45%" convention. Built on `QtQuick.Templates.ProgressBar` with `background` and `contentItem` overridden: background is a pill `Rectangle` (`radius: height / 2`) painted at `Tokens.primary @ 20 % alpha` (shadcn `bg-primary/20`), `clip: true` mirrors shadcn's `overflow-hidden`; contentItem is zeroed (`Item { }`) so the fill lives entirely inside the clipped background region. Two visibility-gated indicator rectangles inside the track — determinate indicator is full track width translated left by `-(1 - visualPosition) * width` (mirrors shadcn's `translateX(-${100 - value}%)` technique; `Behavior on x` at `Tokens.motion.fast` / 150 ms `Easing.OutCubic`, matches shadcn `transition-all`), indeterminate indicator is 40 % track width animated from `-indicator.width` to `track.width` on a 1800 ms `Easing.InOutCubic` infinite-loop `NumberAnimation` (`running:` gate turns animation off when `!indeterminate || !visible`, keeps idle CPU quiet). `focusPolicy: Qt.NoFocus` (non-interactive — no hover / pressed / focus state, no focus ring). Disabled state fades the track `background` to 50 % opacity; root control opacity untouched per pearl-kit rule. Deliberate divergences from shadcn: `indeterminate: bool` mode is pearl-kit-original (shadcn's Progress only renders `value || 0`, so undefined value becomes empty), added for DALI worker-status call sites where completion target is unknown; `to: 100` default instead of `T.ProgressBar`'s `to: 1`; `implicitWidth: 200` instead of shadcn's `w-full` (QML lacks a direct fill-parent equivalent — consumers set `Layout.fillWidth: true` or explicit width).

## [0.2.0] — 2026-04-18

### Changed

- **Breaking**: `PearlKit.Text` renamed to `PearlKit.PearlText`. The previous `Text` identifier shadowed `QtQuick.Text` under an unqualified `import PearlKit 1.0`, forcing consumers into `import PearlKit 1.0 as P` even for typography-only use. `PearlText` avoids the shadow so pearl-kit typography can be mixed with raw `Text` elements without namespace ceremony. Consumers upgrading from `0.1.0`: replace `P.Text { ... }` with `P.PearlText { ... }` (or use the unqualified `PearlText { ... }` if you only import pearl-kit once). The internal atom `PearlKit.internal.PearlText` was renamed to `PearlKit.internal.PearlBaseText` to free the name; this atom is not part of the public API.

### Added

- `PearlKit.ScrollArea` — twelfth public component. Scrollable viewport with thin, rounded auto-hiding scrollbars matching shadcn/ui v4 (`w-2.5` track, `p-px` padding, `rounded-full bg-border` thumb). Built on `QtQuick.Templates.ScrollView` with both `ScrollBar.vertical` and `ScrollBar.horizontal` attached properties replaced by custom `T.ScrollBar` instances (inline `contentItem` Rectangle as the thumb, transparent `Item` background, `minimumSize: 0.1` so the thumb stays grabbable on very long content). Single `autoHide: bool` (default `true`) drives Radix-style fade: scrollbar opacity = 0 when `!control.hovered && !bar.active && !bar.hovered`, else 1, transitioned at `Tokens.motion.base` (180 ms) with `Easing.OutCubic`. Thumb color states — idle `Tokens.border`, hover `Tokens.mutedForeground @ 80 %`, pressed `Tokens.mutedForeground` — transition at `Tokens.motion.fast` (150 ms). Requires `QtQuick.Controls` import alongside `QtQuick.Templates` because the `ScrollBar.*` attached property comes from Controls, not Templates. Deliberate divergences from shadcn: no focus-visible ring on the viewport — `T.ScrollView` silently reparents direct children into its internal `Flickable.contentItem`, which would cause `PearlFocusRing` to scroll with content instead of wrapping the viewport; introducing an Item wrapper to hoist the ring would break `Layout.fillWidth` / attached-property ergonomics, so the ring is dropped (keyboard scrolling feedback is carried by the thumbs themselves); hover / pressed thumb tint is pearl-kit-specific (shadcn ships only `bg-border`) because macOS / Windows desktop cursors expect thumb feedback; `minimumSize: 0.1` is a usability addition not present in shadcn / Radix.
- `PearlKit.Splitter` — resizable panel container for horizontal or vertical split layouts with persistable sizes. Single `withHandle: bool` property toggles a centered 16 × 12 grip-dot rectangle on each handle; default is a bare 1 px hairline in `Tokens.border` with a 6 px drag-hit area. `orientation: Qt.Horizontal | Qt.Vertical` (default `Qt.Horizontal`). Size persistence via convenience `saveLayout()` / `restoreLayout(state)` methods that pass through to Qt's native `T.SplitView.saveState()` / `restoreState()`. Built on `QtQuick.Templates.SplitView` with a custom `handle: Item` delegate — the 1 px-visible + 6 px-hit pattern is realized as an outer Item (hit area) with a centered 1 px Rectangle child (visual). `T.SplitHandle.pressed` / `T.SplitHandle.hovered` attached properties drive the hover-to-`Tokens.ring @ 60 %` transition (`Tokens.motion.fast`, 150 ms, `Easing.OutCubic`). Cursor shape swapped via an interior `MouseArea` with `acceptedButtons: Qt.NoButton` so native drag handling continues to work (pressing the MouseArea is a no-op, events propagate to `T.SplitView`). Focus ring is pearl-kit standard 3 px @ `ring / 50 %` with `offset: 0`, targeting the handle delegate root so Tab-navigation gives per-handle feedback. Orientation inside the handle is read as `control.orientation === Qt.Horizontal` via id reference — `T.SplitView.view` attached property does not resolve on handle delegates (canonical Qt idiom). Supports nesting for VS Code-style group layouts; each nested `Splitter` saves and restores independently. Disabled state fades the hairline and grip to 50 % alpha without touching the root control (standard pearl-kit rule — keeps the focus ring crisp). Deliberate divergences from shadcn's `resizable`: hover tint (`Tokens.ring @ 60 %`, VS Code-pattern; shadcn's default class has no explicit hover) driven by downstream airontgen editor-group use-cases; focus ring geometry (3 px / offset 0) for intra-kit consistency with Button / Input / Toggle / CheckBox instead of shadcn's literal `ring-1 ring-offset-1`; 6 px hit area instead of shadcn's 4 px for more forgiving pointer targeting on touch-screen Macs.
- `PearlKit.FormRow` — eleventh public component. Horizontal label-and-input layout primitive with optional `hint` and `error` text per row. Properties: `label: string`, `hint: string`, `error: string`, `labelWidth: int` (default `120`). Default content slot accepts one child Item (the input); the slot auto-binds `children[0].width = Qt.binding(() => _slot.width)` so the input fills the available column automatically. Root is a plain `Item` wrapping a 2-column `GridLayout` (row 0: label + input, row 1: empty + hint/error at `Layout.column: 1`) because the label must vertically center against row 0 only — a nested `RowLayout` + `ColumnLayout` would drift the label downward when hint/error grows. Reuses `PearlText` with `variant: "label"` (14 px / medium / `leading-none`, shadcn Label parity) for the label and `variant: "muted"` (14 px / regular / `mutedForeground`, shadcn FormDescription parity) for the meta row. When `error.length > 0`, label and meta text recolor to `Tokens.destructive` (shadcn `FormMessage` + `FormLabel data-[error=true]:text-destructive` parity). Color transitions animate at `Tokens.motion.fast` (150 ms). Disabled state fades label + meta text to 50 % alpha and does not auto-propagate to the input child (consumer responsibility, matching shadcn). Deliberate divergences from shadcn: horizontal layout instead of shadcn's vertical stack — desktop form idiom; single primitive instead of shadcn's seven-component `Form` / `FormField` / `FormItem` / `FormLabel` / `FormControl` / `FormDescription` / `FormMessage` tree (pearl-kit has no `react-hook-form` equivalent, so the state-provider layers have no target to bind); no `aria-describedby` auto-wiring (QML uses `Accessible.description` on the input child instead of HTML ARIA); input child width is auto-bound rather than style-driven (`w-full` in shadcn).
- `PearlKit.Card` — tenth public component. Elevated surface primitive — a bordered, padded container with an optional left-border accent stripe for status signaling. Single `variant` property across `default` / `destructive` / `warning` / `success` / `info` drives the accent color; non-default variants auto-enable a 4 px stripe (overridable via `accentWidth`). Root is a plain `Item` (no `QtQuick.Templates` base — a card has no interactive contract). Surface is a child `Rectangle` with `radius: Tokens.radius.xl` (12 px, shadcn `rounded-xl` parity), `border.color: Tokens.border`, `border.width: 1`, and `clip: true` so the accent stripe's left corners round with the card's outer radius (cleanest QML idiom for single-side rounded corners). Content `Column` is a sibling of the surface (not a child, mirrors `GroupBox`) — offset inwards by `padding` on all four sides, children spaced at `spacing` default 24 px for shadcn `flex flex-col gap-6` parity. `padding` and `spacing` overridable per-instance for compact layouts. Disabled state fades background + content column to 50 % alpha; root opacity untouched (standard pearl-kit rule). Deliberate divergences from shadcn: single atomic component instead of shadcn's seven-primitive composition (`Card` + `CardHeader` + `CardTitle` + `CardDescription` + `CardAction` + `CardContent` + `CardFooter`) — `GroupBox` already covers titled/header semantics, and QML `Column` + `Separator` replace the flexbox divider affordance; pearl-kit-original `variant`-driven left-border accent stripe (no shadcn equivalent) motivated by DALI's `ImplantCard` invalid-collision state; no `shadow-sm` (consistency with `GroupBox` — border + background read as elevated on macOS); built-in `Column` layout so `Card { PearlText {} PearlText {} }` just works without a wrapper.
- `PearlKit.Separator` — ninth public component. Thin 1 px divider with `orientation: "horizontal" | "vertical"` (default `"horizontal"`) and `decorative: bool` (default `true`) properties. Root is a plain `Rectangle` with `color: Tokens.border` — no `QtQuick.Templates.*` base because separators have no interactive state. When decorative, `Accessible.role` is `NoRole` so assistive tech skips it; when `decorative: false`, role becomes `Accessible.Separator`. Implicit thickness axis is `1`, non-thickness axis is `0` — consumers set the long axis via `Layout.fillWidth` / `Layout.fillHeight`, anchors, or explicit `width` / `height`. Deliberate divergences from shadcn: pearl-kit does not auto-fill the long axis (shadcn uses Tailwind `w-full` / `h-full`); QML expects container-aware sizing via layouts or anchors, which is the standard Qt pattern.
- `PearlKit.GroupBox` — eighth public component. Titled section container for grouping form rows under a heading, with optional collapse and a built-in "advanced" visibility gate. Properties: `title: string`, `description: string`, `collapsible: bool` (default `false`), `expanded: bool` (default `true`, read-write), `advanced: bool` (default `false`), `advancedVisible: bool` (default `true`). Default content slot lands children in an inner `Column` at 12 px spacing. Collapse runs three concurrent animations (`height`, `opacity` on the inner wrapper, plus outer `Column.spacing`) at `Tokens.motion.base` (180 ms) with `Easing.OutCubic`; the wrapper is `clip: true` so children don't leak during the height transition. Visibility gate computes `visible: !advanced || advancedVisible` so `advanced && !advancedVisible` collapses the layout slot entirely (DALI `_mark_advanced` parity). Root is a plain `Item` (not `T.GroupBox`) for full control over the collapse animation. Reuses `PearlText` (title / description), `PearlChevron` (rotates 180° on expand via `Behavior on rotation`), and `PearlFocusRing` (3 px @ `ring/50`, `offset: 2`, visible only when `collapsible && activeFocus`). Header click region is an inline `MouseArea` with `PointingHandCursor`; keyboard activation via `Keys.onSpacePressed` / `onReturnPressed` on the root, gated by `collapsible`. Disabled state fades header + body separately, never the root, so the focus ring stays crisp. Deliberate divergences from shadcn: pearl-kit-original component (no shadcn `GroupBox` primitive — composed from shadcn `Card` visuals + `Accordion` collapse semantics + DALI advanced-settings gate); `Tokens.radius.lg` (8 px) instead of shadcn `Card`'s `rounded-xl` (12 px) for a tighter form-section feel; no `shadow-sm` — settings sections sit flat against the tab background; no header-action or footer slot (deferred).

## [0.1.0] — 2026-04-18

### Added

- `PearlKit.Button` — first public component. Six variants (`default`, `destructive`, `outline`, `secondary`, `ghost`, `link`) x five sizes (`default`, `xs`, `sm`, `lg`, `icon`) with `iconLeft` / `iconRight` URL slots. Built on `QtQuick.Templates.Button` with state-reactive background and foreground color transitions, keyboard-only focus ring (3px, 50% ring-color alpha, no offset), and shadcn/ui v4 visual parity.
- `PearlKit.Input` — second public component. Single 36 px size (shadcn parity — Input has no size variants), `property bool error` for invalid-state visuals (border + ring switch to destructive), `iconLeft` / `iconRight` URL slots (pearl-kit extension over shadcn for Button consistency), theme-aware background (transparent in light mode, `Tokens.input` at 30% alpha in dark modes). Built on `QtQuick.Templates.TextField` with the default `TextInput` contentItem preserved for native caret, IME, selection, and undo/redo support. Focus ring appears on any focus source (mouse or keyboard) rather than `visualFocus`-only as Button — text inputs need caret feedback on click too.
- `PearlKit.Toggle` — third public component. iOS-style switch with two sizes (`sm`, `default`) matching shadcn/ui v4 Switch geometry (24×14 and 32×18 px tracks, 12 and 16 px thumbs). Built on `QtQuick.Templates.Switch` with a custom `indicator` slot (rounded-full track + animated thumb). Keyboard-only focus ring (3 px @ `ring/50`, offset 0) targets `control.indicator` so the ring follows the rounded-full track radius instead of the default rectangular fallback. Thumb position animated via `Behavior on x` (NumberAnimation) and color via `Behavior on color` (ColorAnimation), both at `Tokens.motion.fast` with `Easing.OutCubic`. Named `Toggle` rather than shadcn's `Switch` for pearl-kit roadmap and DALI migration consistency. Thumb color uses a uniform `foreground` / `primaryForeground` formula across all themes instead of a `Tokens.isLight`-branched approach — deliberately higher-contrast in Light mode than shadcn. Track unchecked background uses full-alpha `Tokens.input` across all modes (shadcn uses `dark:/80`). No `shadow-xs` (skipped by design, negligible on a control this small).
- `PearlKit.Select` — fourth public component. Non-editable dropdown with two sizes (`sm`, `default`), `property bool error` for invalid-state visuals, model-driven API (supports string arrays, ListModel, and JS object models via `textRole` / `valueRole`), popper-positioned popup with fade + scale enter transition (`Tokens.motion.fast`), and `type`-role grouped models (`"item"` / `"header"` / `"separator"`) as a unified replacement for shadcn's `SelectLabel` + `SelectSeparator` + `SelectGroup` primitives. Built on `QtQuick.Templates.ComboBox` with every slot overridden: `background` (Input-style rectangle), `contentItem` (PearlText with placeholder handling), `indicator` (new `PearlChevron` internal atom), `popup` (inline `T.Popup` parented to `T.Overlay.overlay`), and `delegate` (`T.ItemDelegate` branching on the row type). Commits to header or separator rows are guarded by an `onActivated` interceptor that reverts to the last valid selection. Keyboard navigation (Up / Down / Home / End / Space / Return / Escape / type-ahead) is handled natively by `T.ComboBox`; note that Qt's built-in nav does not skip non-selectable rows, so the keyboard highlight passes through headers and separators — the commit guard handles this cleanly. Deliberate divergences from shadcn: popper-only positioning (no item-aligned), static 320 px popup max height, no ScrollUp / ScrollDown chevron buttons (native `ScrollIndicator` used instead), minimal filled-dot current-row indicator instead of a check glyph, no `shadow-md` on the popup.
- `PearlKit.internal.PearlChevron` — Canvas-based stroked chevron atom with `strokeColor`, `strokeWidth`, and `direction` (`"down"` / `"up"`) properties. Avoids adding a new Qt.labs dependency for Select's indicator. Re-usable for future components that need a chevron glyph.
- `PearlKit.Dialog` — fifth public component. Modal popover with `title`, `description`, default body slot, `footer: Component`, `showCloseButton`, `maxWidth`, and `initialFocusItem` properties. Built on `QtQuick.Templates.Popup` (not `T.Dialog` — `T.Dialog`'s `standardButtons` + `accepted()` / `rejected()` ecosystem is tied to a fixed Ok / Cancel taxonomy that shadcn does not use). Overlay is a 50 %-black `T.Overlay.modal` `Rectangle` with a 200 ms fade via `Behavior on opacity`. Enter / exit transitions run parallel `opacity` (0 → 1) and `scale` (0.95 → 1.0) `NumberAnimation`s over 200 ms with `Easing.OutCubic`, matching shadcn's `fade-in-0 zoom-in-95` / `fade-out-0 zoom-out-95`. Background shadow uses `QtQuick.Effects.MultiEffect` (Qt 6 official path; `Qt5Compat.GraphicalEffects.DropShadow` is deprecated). Close button is a 16×16 `Rectangle` with a new `PearlXIcon` Canvas atom centered inside and a 2 px focus ring at 2 px offset. Body slot uses `default property alias _bodyContent: bodyColumn.data` — consumer children land in a 16 px-spaced body Column. Footer is a `property Component` rendered through a `Loader` for maximum composition flexibility. Positioning is manual `Math.round((parent.dim - own.dim) / 2)` since `T.Popup` does not expose `anchors`. Deliberate divergences from shadcn: single component instead of five separate React primitives, `T.Popup` base instead of `T.Dialog`, single-layer shadow, `maxWidth` as an int property instead of a Tailwind responsive class.
- `PearlKit.internal.PearlXIcon` — Canvas-based stroked X atom with `strokeColor`, `strokeWidth` properties. Mirrors the `PearlChevron` pattern; avoids shipping an SVG asset or taking a dependency on Lucide / Qt5Compat. Reusable by any future component needing an X glyph (close buttons, dismiss badges, etc.).
- `PearlKit.CheckBox` — sixth public component. 16×16 rounded-square box with a lucide-style checkmark glyph on checked, horizontal dash on indeterminate. Single size (shadcn parity). Built on `QtQuick.Templates.CheckBox` with `property bool error` (mirrors `Input` / `Select` patterns) that switches border + focus ring to `destructive` color. `indicator` slot overridden as a `Rectangle` with the check glyph centered inside; `contentItem` zeroed out to force indicator-only rendering (label composition is the consumer's responsibility, matching shadcn). Focus ring 3 px @ `ring / 50 %`, keyboard-only via `visualFocus`, target `control.indicator` so the radius wraps the 4 px box correctly. No `Behavior on color` — check / uncheck snaps instantly per shadcn's `transition-none` directive. Deliberate divergences from shadcn: `tristate` / indeterminate support (Radix primitive exposes it but shadcn doesn't stylize), no built-in label slot (external compose like shadcn). Error ring alpha is theme-aware (0.2 light / 0.4 dark) matching shadcn's `dark:ring-destructive/40`.
- `PearlKit.internal.PearlCheckIcon` — Canvas-based stroked check / dash atom with `strokeColor`, `strokeWidth`, and `mode` (`"check"` / `"dash"`) properties. Normalized coordinates scale to any size; `onWidthChanged` / `onHeightChanged` repaint ensures sharp render at non-default dimensions. Mirrors `PearlChevron` / `PearlXIcon` pattern. Reusable by any future component needing a check or dash glyph.
- `PearlKit.PearlText` — typography primitive (Tier 0 P0, most-used widget). Single `variant` property — `title`, `heading`, `body` (default), `muted`, `label`, `code`, `mono` — drives font family, size, weight, color, and line height from `Tokens.font.*` and `Tokens.foreground` / `Tokens.mutedForeground`. Root is a pure `QtQuick.Text` Item (no `Templates` base, no `Rectangle` wrapper) so every native property passes through — `elide`, `wrapMode`, `horizontalAlignment`, `maximumLineCount`, `textFormat`. `renderType: Text.NativeRendering` + `antialiasing: true` explicit for crisp Retina rendering. Disabled state fades color to 50 % alpha; font properties unchanged. Named `PearlText` rather than `Text` so it does not shadow `QtQuick.Text` under an unqualified `import PearlKit 1.0` — consumers can mix pearl-kit typography with raw `Text` elements without namespace ceremony. Deliberate divergences from shadcn: single component instead of `<Label>` + Tailwind typography classes (`variant: "label"` covers shadcn Label parity, `text-sm leading-none font-medium`); `code` variant renders mono-muted text without shadcn's `bg-muted` chip (pure `Text` Item cannot render a background — consumers wrap in a `Rectangle` if needed); `mono` variant is pearl-kit-specific (mono + foreground for data display vs. `code`'s mono + muted for documentation flavor); shadcn typography `h1..h4` / `lead` / `large` / `small` variants not shipped — not requested by DALI call-sites, can be added on demand.
- `PearlKit.Stepper` — seventh public component. `[−] value [+]` layout with dedicated decrement / increment buttons at the track edges. Single `value: real` API across integer and floating-point modes via a hidden integer-scaling layer (`decimals: 0` behaves as int; `decimals: 1..3` preserves the float precision by storing `Math.round(value * 10^decimals)` in the internal `T.SpinBox.value` int). Root is an `Item` wrapper around `T.SpinBox` because `T.SpinBox.value`'s int typing cannot be overridden in-place. Visual composition mirrors `Input` (36 px track, `Tokens.radius.md`, 1 px `Tokens.border`, theme-aware background) with two 32 px ghost-Button-style edge indicators (`Tokens.accent` hover tint at 40% / 60%). Edge buttons auto-disable when `value` hits `from` or `to` (DALI parity). `suffix` (e.g. `"%"`, `"px"`) and `specialValueText` (shown when `value == from`, e.g. `"Auto"`) bake into `textFromValue` / `valueFromText`; on focus-in, the line edit shows the raw value (no suffix) and `selectAll()` runs, on focus-out the format is re-bound. `Keys.onEscapePressed` restores the bound display and blurs. Focus ring 3 px @ `ring / 50 %` on any focus source (`activeFocus`, Input pattern), destructive-tinted when `error: true`. Validator swaps between `IntValidator` and `DoubleValidator` based on `decimals`. Derivative from shadcn primitives — shadcn/ui does not ship a stepper; behavioral reference is DALI's `DaliStepper` / `DaliDoubleStepper` (~20 call sites for zoom, dimensions, thresholds). Deliberate divergences documented: unified single-API real-valued `value`, `decimals` capped at 3 to avoid 32-bit overflow in the integer-scaled SpinBox, no `wrap`, ghost-button hover tint on the edge buttons.
- `PearlFocusRing` parameterized: new `ringWidth` (default 2) and `ringColor` (default `Tokens.ring`) properties for per-component ring customization. Backward-compatible with existing defaults.
